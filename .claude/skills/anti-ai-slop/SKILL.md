# Anti-AI-Slop Gate (Проверка Содержательности)

Skill отвечает на один вопрос: можно ли перенести этот вывод в другой продукт, ничего не потеряв? Если да — это AI slop, и артефакт получает `needs_revision`. Gate проверяет качество результата, а не список слов: артефакт без единого запрещённого слова всё ещё может быть slop.

**Полная процедура, тесты и failure modes — в [`agent-pack/skills/anti-ai-slop/SKILL.md`](../../../agent-pack/skills/anti-ai-slop/SKILL.md). Следуй ей.** Нормативный текст — [`agent-pack/workflows/claude-operating-rules.md`](../../../agent-pack/workflows/claude-operating-rules.md), раздел 3.

## Когда использовать
- Перед записью research pack, CJM, opportunity roadmap, PRD, copy deck.
- Перед любой публикацией в Notion/Figma (жёсткий gate: провал lint запрещает external write).
- На `11-qa` при аудите содержательности артефактов.

## Ключевые шаги
- `yarn research:lint <run-dir>` — исполняемая проверка Rules 1-6.
- Тест переносимости: подставь другой продукт из другой отрасли; вывод остался верным — в нём нет доменной детали.
- Тест механизма: «повысит доверие»/«снизит friction» допустимы только с механизмом, моментом пути и метрикой.
- Метрики (conversion, retention, trust) раскрываются как конкретное действие пользователя или бизнеса.
- Roadmap/ICE/RICE ссылаются на конкретное трение из CJM и способ проверки.
- Skill правит формулировки, но не дописывает факты: пустой вывод из-за отсутствия источника — это `needs_validation`.

## Обязательные проверки
- `yarn research:lint <run-dir>`
