# Frontend Result

## Что реализовано

Добавлен отдельный прототип портфолио по route family:

```text
/portfolio
/portfolio/:companyId
/portfolio/:companyId/case/:caseId
```

Реализация находится в:

- `apps/frontend/src/App.tsx`
- `apps/frontend/src/views/PortfolioView.tsx`
- `apps/frontend/src/styles.css`

## Содержательная структура

Главная:

- показывает только компании;
- не показывает отдельные кейсы;
- использует editorial index: номер, компания, сфера, роль, период, количество кейсов.

Страница компании:

- показывает контекст компании;
- показывает роль и команду;
- выводит список кейсов этой компании.

Детальная страница кейса:

- контекст;
- проблема;
- решение;
- результат;
- схематичное preview интерфейса.

## Что взято из Figma Make

- Теплый фон.
- Serif display typography.
- Mono metadata labels.
- Строки-индексы вместо карточной витрины.
- Company page как case index.
- Case page как editorial article.

## Что изменено относительно Make

Make сгенерировал часть контента сам. В локальном прототипе:

- удалены выдуманные кейсы вроде `mobile-app` для А3;
- заменены фантазийные метрики на данные/формулировки из реальных кейсов;
- сохранен правильный flow пользователя: `главная -> компания -> кейсы -> кейс`.
