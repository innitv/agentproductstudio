# Навык: Figma Design Token Extractor

## 1. Context (Контекст)
Для обеспечения 100% визуального соответствия (Bespoke UI) фронтенд не должен писать "абы какие" цвета или стили. Этот навык описывает пошаговый процесс извлечения дизайн-токенов из утвержденных макетов Figma, маппинга их на CSS-переменные в `styles.css` и верификации с общей картой дизайн-системы.

## 2. Triggers (Триггеры)
Агент применяет этот навык на этапе:
- **Стадия воркфлоу**: `04-design` (проектирование концепции) и `08-frontend` (реализация интерфейса).
- **Событие**: Получение visual reference или требований сверстать кастомный дизайн.
- **Вход**: Ссылка на файл Figma в `recursive-brief.md` или `run-plan.md` и настроенный Figma MCP-сервер.

## 3. Action Step-by-Step (Алгоритм выполнения)

### Шаг 1: Чтение и парсинг слоев Figma
1. Запустить подключение к Figma API через Figma MCP-сервер (`use_figma` или `figma_read`).
2. Считать стили документа (styles) и переменные (variables) из целевого файла макета.
3. Скачать изображения иконки и логотипа при необходимости.

### Шаг 2: Извлечение визуальных токенов
1. **Цвета (Colors)**: Извлечь все HEX-значения для основных (Primary), вторичных (Secondary), нейтральных (Neutral) цветов и состояний наведения/фокуса.
2. **Типографика (Typography)**: Записать размеры шрифтов (Font sizes), межстрочные интервалы (Line heights) и начертания (Font weights).
3. **Эффекты (Shadows & Radius)**: Записать тени блоков (box-shadows) и скругления углов (border-radius).

### Шаг 3: Генерация и компиляция токенов
1. Сверить полученные токены с общей дизайн-библиотекой в `design/figma/a3-design-system/token-map.md`.
2. Сгенерировать CSS-переменные в блоке `:root` в `apps/frontend/src/styles.css`:
   ```css
   :root {
     --primary-color: #005ffc;
     --bg-card: rgba(15, 23, 42, 0.6);
     --border-radius-lg: 16px;
   }
   ```

### Шаг 4: Запись дизайн-брифа
1. Записать экспортированные токены в `design-brief.md` в секцию `## Visual Direction`.

## 4. Validation / Quality Gates (Критерии качества)
- [ ] Все ключевые цвета (Primary, Secondary, Background) извлечены без потерь и записаны в CSS.
- [ ] Относительные значения отступов (Spacing) и скруглений соответствуют сетке (например, кратной 4px или 8px).
- [ ] Переменные успешно импортируются и распознаются в `App.tsx`.
