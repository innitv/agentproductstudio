## Primary Screen

**Главный экран оплаты (Payment Checkout / Dashboard)**: единая точка сборки платежного контекста. Включает блок выставленного счета, выбор метода оплаты (СБП, карта, BNPL) и детализацию платежа.

## Primary Action

**Подтверждение и совершение платежа**: кнопка "Оплатить" (с указанием выбранного метода СБП/BNPL/карта и итоговой суммы с учетом комиссий).

## Completion Step

**Экран успеха с чеком и статусом платежа**: показывает финальный статус, чек с деталями транзакции, кнопку скачивания PDF и трекер для возможного возврата (сценарии e-commerce/услуг).

## Primary User Flow

1. Пользователь переходит по ссылке выставленного счета (ЖКХ или корзина магазина).
2. Проверяет детали платежа (получатель, сумма, назначение).
3. Выбирает метод оплаты (СБП для экономии на комиссии или BNPL для оплаты частями).
4. Нажимает "Оплатить" -> Система обрабатывает платеж.
5. Открывается страница статуса с подтверждением "Оплачено" и возможностью скачать чек.
6. При необходимости вернуть деньги (например, отмена услуги) пользователь переходит в историю платежей и инициирует возврат.
7. Пользователь отслеживает этапы на экране трекера возврата.

## Primary Flow Tree

```text
Переход по счету
└─ Отображение деталей счета и селектора методов оплаты (Экран оплаты)
   └─ Выбор способа (СБП / BNPL / Карта) -> Нажатие "Оплатить"
      ├─ Успешный платеж -> Перенаправление на Экран статуса (Успех) -> Доступен чек и кнопка "Оформить возврат"
      └─ Ошибка платежа -> Отображение ошибки на Экране оплаты -> Предложение сменить метод оплаты
```

## Decision & Friction Map

| Moment | User risk / question | IA response | Required proof / copy / state |
|---|---|---|---|
| Выбор СБП | Будет ли комиссия? | Показать плашку "СБП — 0% комиссии" | Текст: "Оплата через СБП без комиссии со стороны A3 Pay" |
| Выбор BNPL | Это кредит? Какие скрытые переплаты? | Показать график платежей | График: "4 платежа по 25%, первый сегодня, без процентов" |
| Запуск возврата | Куда вернутся деньги? Когда? | Объяснить логику возврата на исходное платежное средство | Текст: "Деньги вернутся на карту/счет, с которого производилась оплата" |

## State Map

| State | Where it appears | User need | Required copy / UI behavior |
|---|---|---|---|
| Default | Экран оплаты (`/checkout`) | Выбрать способ оплаты и проверить детали | Заполненные поля, активный селектор способов |

## Navigation Rules

| Navigation element | Purpose | Visibility | Mobile behavior | Rule |
|---|---|---|---|---|
| Top nav | Логотип, кнопка "Назад" к мерчанту | Всегда вверху | Ужимается до логотипа и компактной стрелки | Должен позволять вернуться к покупкам без потери сессии |
| Bottom sheet | Выбор банка в СБП / Сведения о BNPL | При выборе соответствующего метода | Выезжает снизу во весь экран | Удобно для выбора банка из списка в один тап на мобильных |
| Footer | Ссылки на условия сервиса, поддержку и лицензии | Внизу страницы | Сворачивается в стек | Обеспечивает юридическое доверие (REQ-002) |

| Element | Requirement |
|---|---|
| H1 | Единственный заголовок страницы: "Оплата счета" или "Статус платежа" |
