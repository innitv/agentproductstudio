# Reference Analysis

## Inputs Used

- User request: "Необходим сайт для продажи валорант поинт. Референс этот сайт https://pixelperfect.school/ai-native-designer"
- `prd.md`
- `ia-brief.md`
- Riot legal/support constraints from `research-summary.md`

## References

| Reference | Type | What user expects | Legal/IP risk | Evidence / source |
|---|---|---|---|---|
| Pixel Perfect AI Native Designer | education/course sales page | Long-form conversion structure, large type, strong CTA rhythm | medium: avoid direct copy of layout, copy, brand and distinctive composition | User reference |
| VALORANT/Riot prepaid support docs | official support | Redeem/region facts | low: factual reference only | Riot support |

## Allowed Patterns

| Pattern | Why allowed | How to adapt originally |
|---|---|---|
| Oversized editorial typography | Common landing/sales pattern | Use VP Nexus copy hierarchy and product-specific headings |
| Numbered section rhythm | Common education/sales page pattern | Turn it into region -> package -> flow -> trust -> FAQ |
| Dense pricing/program cards | Common commerce pattern | Use VP package cards with region labels |
| Strong CTA and section anchors | Common landing pattern | Region-first CTA instead of copied official CTA |
| Region/account safety messaging | Source-backed functional need | Use official facts in plain language with citations |

## Disallowed Copying

| Element | Why disallowed | Replacement direction |
|---|---|---|
| Pixel Perfect text, course claims, exact section order | Copyright/trade dress risk | Write original VP marketplace copy and order |
| Pixel Perfect brand, logo, instructors, screenshots | Brand confusion | Use VP Nexus name and original generated asset |
| Exact colors, spacing and component composition | Trade dress risk | Use independent cream/black/red/cyan commerce system |
| Riot/VALORANT logos | Trademark/affiliation risk | Text disclaimer and VP Nexus brand |
| Official agent/key art/maps/screenshots | Copyrighted game assets | Generated abstract prepaid code cards |
| Official typography/color system as exact clone | Brand identity copying | Custom editorial commerce palette |

## Design Implications

- Layout: sales landing with region-first commerce flow.
- Color: light editorial base, black structure, red urgency accent, cyan verification accent.
- Typography: system font, large H1 and section headings.
- Imagery: generated abstract code-card render, no Riot assets.
- Interaction: package selection, region chips and CTA anchors.
- Accessibility: maintain contrast, semantic headings and native controls.

## Decision

Use Pixel Perfect as structural inspiration only. Build an original independent marketplace with explicit disclaimer and no Riot/Pixel Perfect asset copying.
