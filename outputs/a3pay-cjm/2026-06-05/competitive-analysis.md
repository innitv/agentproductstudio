## Competitor Set

## Конкурентный контекст

A3 Pay не стоит сравнивать только с кошельками. Реальная конкурентная карта состоит из пяти слоев:

1. Платежные маршруты: СБП, карточный эквайринг, Мир, QR, NFC/BLE, биометрия.
2. Банковские кнопки оплаты: SberPay, T-Pay, платежи в банковских приложениях.
3. Кошельки, предоплаченный баланс и кошельки маркетплейсов: ЮMoney, экосистемы O

## Набор конкурентов

- СБП / СБПэй.
- SberPay / T-Pay / банковские кнопки оплаты.
- ЮMoney и кошельки маркетплейсов.
- Провайдеры BNPL.
- Госуслуги / ГИС ЖКХ / банковские каталоги счетов.
- Классический карточный эквайринг.


## Comparison Matrix

## Матрица позиционирования

| Тип игрока | Сильная сторона | Слабая сторона | Возможность для A3 Pay |
|---|---|---|---|
| СБП / СБПэй | Массовый платежный маршрут, привычное поведение вокруг номера телефона, низкая стоимость для продавца. | Не закрывает весь жизненный цикл: напоминания, корзину, споры, статус многостороннего платежа. | Быть UX-слоем оркестрации поверх платежного маршрута, не конкурировать с ним. |
| Банковские кнопки оплаты | Доверие, сохраненные инструменты, высокая ежедневная частота. | Замкнуты на банковскую экосистему, ограниченная межбанковская нейтральность. | Нейтральный платежный запрос по номеру телефона между пользователем, продавцом и поставщиком услуги. |
| Кошельки маркетплейсов | Сильны внутри закрытой коммерции, loyalty и BNPL-связок. | Ограничены доменом маркетплейса. | Регулярные и сервисные платежи между доменами. |
| Госуслуги / ГИС ЖКХ | Официальные данные, доверие к государственному сервису. | UX фрагментирован по ведомствам, возвраты/статусы не всегда удобны для пользователя. | Статус платежа, напоминания, семейные платежи, слой поддержки. |
| Провайдеры BNPL | Рост конверсии, высокий средний чек. | Регулирование, скоринг риска, концентрация рынка. | BNPL-брокер и выбор способа оплаты в checkout. |
| Классический эквайринг | Универсальное принятие, зрелые возвраты. | Трение ввода карты, комиссии продавца, слабая идентичность по телефону. | Мультимаршрутный резервный сценарий с phone-first идентичностью. |


## Takeaways

## Активы A3

| Актив | Доказательство | Стратегическое применение |
|---|---|---|
| Существующий B2B-платежный сервис и сеть поставщиков | Официальный сайт A3 указывает, что компания помогает банкам проводить платежи в пользу 60k+ сервисных компаний. | Использовать каталог поставщиков и банковские партнерства для быстрого расширения направления оплаты счетов. |
| Домен регулярных платежей | Публичное позиционирование A3 подчеркивает оптимизацию регулярных/сервисных платежей. | Сделать контроль регулярных платежей одной из центральных продуктовых тем. |
| Платежные ссылки / формы / СБП в контексте оферты | Публичная оферта A3 упоминает платежные ссылки и операции СБП. | Строить сценарии запроса оплаты по телефону, а не только интерфейс чистого кошелька. |
| Доказательства по банковским партнерам | Tavily-скан официального сайта A3 показал отзывы банков про расширение базы поставщиков, A3.Notification, A3.Household и рост транзакционно активных клиентов в партнерских приложениях. | Приоритизировать B2B2C-выкатку через банковские приложения и каталоги поставщиков, а не самостоятельное привлечение пользователей первым шагом. |

## Незакрытые конкурентами разрывы

- Единый центр управления платежными обязательствами пользователя: счета, подписки, бронирования поездок и поэтапные покупки.
- Запрос оплаты по номеру телефона для доверенных отношений: семья, арендодатель, туристический агент, брокер по авто, поставщик услуг.
- Доказательство статуса платежа: оплачено, ожидает подтверждения, возвращено, принято поставщиком, подтверждено реестром или госстатусом.
- Разделение многостороннего платежа и поэтапные платежи, где один заказ включает продавца, госпошлину, страховку, доставку, брокера и поставщика услуг.

## Стратегическая рекомендация

Не запускать A3 Pay как “еще один кошелек”. Запускать как контрольный слой A3 Pay:

- платежный запрос по номеру телефона;
- умная платежная корзина;
- выбор платежного маршрута: СБП, карта, кошелек, BNPL, рассрочка;
- менеджер регулярных поручений;
- центр подтверждений и статусов;
- партнерские API для поставщиков, банков и сервисных платформ.
