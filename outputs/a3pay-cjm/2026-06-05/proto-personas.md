## Proto Personas

## Персона 1: Регулярный плательщик семьи

| Поле | Описание |
|---|---|
| Контекст | Оплачивает ЖКХ, связь, штрафы, кружки, подписки и иногда счета родственников. |
| Цели | Не забывать платежи, видеть все обязательства, быстро подтверждать оплату. |
| Боль | Разные приложения, разные реквизиты, непонятно что уже оплачено, сложно делегировать платеж. |
| Роль A3 Pay | Умная корзина счетов, напоминания, семейный плательщик, подтверждение оплаты, платежный запрос по телефону. |
| Метрика успеха | Уменьшение просрочек, рост повторных платежей, снижение обращений в поддержку по статусу. |

## Персона 2: Покупатель крупного товара или авто

| Поле | Описание |
|---|---|
| Контекст | Покупает авто, технику, мебель или сервис с несколькими платежными этапами. |
| Цели | Понимать полную стоимость, платить по этапам, не ошибиться в реквизитах. |
| Боль | Предоплаты, доплаты, пошлины/сборы, страхование, доставка, разные получатели. |
| Роль A3 Pay | Платежные этапы, разделение по получателям, калькулятор, эскроу-подобный статус, хранилище квитанций. |
| Метрика успеха | Конверсия в оплату этапа, снижение брошенных платежей после счета, снижение доли споров. |

## Персона 3: Цифровой путешественник

| Поле | Описание |
|---|---|
| Контекст | Покупает билеты, отели, страховки, eSIM, экскурсии, доплаты за багаж/места. |
| Цели | Не терять платежи и возвраты, быстро оплачивать допуслуги. |
| Боль | Разные провайдеры, валютные/локальные ограничения, отмены и возвраты, дедлайны. |
| Роль A3 Pay | Платежный маршрут поездки, напоминания от поставщиков, статус возвратов, оплата по ссылке или телефону. |
| Метрика успеха | Доля подключенных допуслуг, успешность платежей, NPS по возвратам. |

## Персона 4: Продавец или поставщик услуги

| Поле | Описание |
|---|---|
| Контекст | Поставщик услуг, управляющая компания, туристический агент, брокер, сервисный бизнес. |
| Цели | Получать оплату быстро, меньше ошибок в назначении платежа, выше повторяемость платежей. |
| Боль | Ручное сопоставление платежа, неоплаченные счета, дорогая поддержка по статусам. |
| Роль A3 Pay | Запрос оплаты по телефону, каталог поставщиков, идентификатор сверки, обратные вызовы по статусу. |
| Метрика успеха | Доля завершенных платежей, уровень автоматической сверки, срок получения оплаты. |

## Персона 5: Банк или финтех-партнер

| Поле | Описание |
|---|---|
| Контекст | Банк хочет удерживать клиента в своем приложении и расширять каталог платежей. |
| Цели | Больше транзакций, ниже стоимость обслуживания, релевантные платежные сервисы. |
| Боль | Сложно подключать тысячи поставщиков, удерживать качество данных и UX. |
| Роль A3 Pay | White-label платежный каталог, интеграция поставщиков, регулярные сервисы, аналитика. |
| Метрика успеха | Транзакции на активного пользователя, покрытие сервисов, платежная выручка, рост удержания. |


## Decision Context

Персоны используются для выбора P0/P1 сценариев, CJM-путей и скрипта реальных интервью. Они не являются подтвержденными сегментами рынка до пользовательской валидации.

## Validation Plan

- Провести интервью по регулярным платежам, авто/крупным покупкам, travel, поставщикам услуг и банковским партнерам.
- Проверить готовность платить по номеру телефона, доверие к проверенному получателю и ценность статуса после оплаты.
