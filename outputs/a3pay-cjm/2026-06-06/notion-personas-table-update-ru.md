# Прото-персоны A3 Pay: табличное обновление для Notion

Статус: подготовлено локально, не опубликовано.

Это обновление нужно использовать для замены или дополнения раздела `03 Прото-персоны` в опубликованном Notion hub. Внешняя запись требует нового approval через `workflow:approval-request`.

## Вопросы для проверки по персонам

| Персона | Что проверить интервью | Минимальный сигнал |
|---|---|---|
| Регулярный плательщик семьи | Какие обязательные платежи чаще всего забываются или вызывают страх ошибки | 5-7 интервью с повторяющимися pains по счетам/статусам |
| Контролирующий подписки | Готов ли пользователь переносить управление подписками в A3 Pay | 5 интервью + список сервисов, где нужен mandate control |
| Организатор поездки | Есть ли боль в split/refund/status при travel payments | 5 интервью с recent travel payments |
| Покупатель или владелец автомобиля | Какие участники авто-сделки требуют verified payment request | 5 интервью с покупателями/владельцами авто и 2-3 expert calls |
| Merchant / сервисный бизнес | Снижает ли A3 Pay failed payments или ручную сверку | 5 merchant interviews + данные по отказам/возвратам |

## Решение для Notion layout

| Блок | Формат в Notion | Причина |
|---|---|---|
| Таблица прото-персон | table | сравнение сегментов, JTBD, pains и value proposition |
| Матрица приоритета | table | связь персон с MVP/discovery |
| Вопросы для проверки | table или database-ready table | превращает гипотезы в research backlog |
| Детали по каждой персоне | selective toggles | раскрывать только при необходимости, не перегружать страницу |
